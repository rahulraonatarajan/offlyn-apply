🧪 Dropdown vs Text Field Detection Test

How to Test

  1. Load this page with the Offlyn extension enabled
  2. Click the Offlyn popup to open the field detection panel
  3. Check each field's icon and type:
    • [T] = Text field
    • [▼] = Dropdown (select)
    • [▼?] = Autocomplete dropdown
    • [E] = Email field
    • [P] = Phone field
  4. Dropdowns should display: "Options: option1, option2, option3..."
  5. Try the "Auto-fill" button and verify:
    • Text fields: Values are typed in
    • Dropdowns: Options are selected from the list

📝 Text Fields (Should show [T] icon)

Expected: Icon [T], Type: text, No options shown
Expected: Icon [E], Type: email, No options shown
Expected: Icon [P], Type: tel, No options shown

📋 Dropdown Lists (Should show [▼] icon)

Expected: Icon [▼], Type: select-one, Options listed
Expected: Icon [▼], Type: select-one, Options listed
Expected: Icon [▼], Type: select-one, Options listed
Expected: Icon [▼], Type: select-one, Options listed

📝 Multi-line Text (Should show [A] icon)

Expected: Icon [A], Type: textarea, No options shown

✅ Success Criteria

  • Text fields show [T] icon and type "text"
  • Email fields show [E] icon and type "email"
  • Phone fields show [P] icon and type "tel"
  • Dropdown fields show [▼] icon and type "select-one"
  • Dropdown fields display "Options: option1, option2..."
  • Textarea shows [A] icon and type "textarea"
  • Auto-fill correctly types into text fields
  • Auto-fill correctly selects from dropdowns (not typing)
  • No console errors during detection or filling
